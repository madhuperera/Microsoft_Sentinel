### 🧩 10. Activity types: what counts and what is background noise

| Workbook context | Value |
|---|---|
| Selected period | `{W_TimeRange:start}` to `{W_TimeRange:end}` |

This view lists every type of activity seen in the period and shows whether it is counted as meaningful staff activity or treated as background and system noise. It is mainly a check that the other views are focusing on the right activity.

This is a configuration and quality check. It is not a security view and not a performance view.

#### Column guide

| Column | What it means |
|---|---|
| `Workload` | The Microsoft 365 app. |
| `Operation` | The specific action recorded. |
| `Category` | A plain grouping for the action, for example email, file authoring, file access or sharing. |
| `EngagementSignal` | Whether the action is counted as meaningful staff activity (see below). |
| `Events` | How many times the action occurred in the period. |
| `Users` | How many different people performed the action. |

#### Meaning of `EngagementSignal`

| `EngagementSignal` | Meaning |
|---|---|
| `Yes` | Counted as meaningful staff activity in the other views. |
| `No` | Treated as background or system noise, and not counted. |
| `UNMAPPED` | A new action that has not been classified yet. Worth assigning to a category. |

#### How rows are sorted

Unclassified actions appear first, then the highest event counts, so anything new or high-volume stands out.
