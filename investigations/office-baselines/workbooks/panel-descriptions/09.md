### 🔎 09. The detail behind one person's numbers

| Workbook context | Value |
|---|---|
| Selected period | `{W_TimeRange:start}` to `{W_TimeRange:end}` |
| Person shown | `{W_TargetUser}` *(required; the view is empty until set)* |

This view lists the individual activity records for one person, most recent first. Use it to understand what actually sits behind a number in the other views before drawing any conclusion. The view stays empty until a person is chosen.

This is detailed personal data. Open it only when there is a clear and proportionate reason, and handle it carefully.

#### Column guide

| Column | What it means |
|---|---|
| `TimeGenerated` | When the activity happened. |
| `UPN` | The person's sign-in address. |
| `OfficeWorkload` | The Microsoft 365 app, for example Exchange, SharePoint, OneDrive or Microsoft Teams. |
| `Operation` | The specific action recorded, for example a file upload or a message sent. |
| `RecordType` | The category of audit record the action belongs to. |
| `ResultStatus` | Whether the action succeeded. |
| `ClientIP` | The network address the action came from. |
| `UserAgent` | The browser or app used. |
| `OfficeObjectId` | The item involved, such as a file or site path. |
| `SourceFileName` | The file name, where the action involved a file. |
| `Site_Url` | The SharePoint or OneDrive site address, where relevant. |

#### What is included

Everyday staff activity for the chosen person only. Some columns may be blank when they do not apply to a particular action.

#### How rows are sorted

Rows show the most recent activity first.
