### 🗂️ 02. Each person's change, app by app

| Workbook context | Value |
|---|---|
| Selected period | `{W_TimeRange:start}` to `{W_TimeRange:end}` |
| Baseline used for comparison | The immediately previous period of the same length |
| Normal change band | `+/- {W_NormalBandPct}%` |
| Target user filter | `{W_TargetUser}` *(blank means all users)* |

This view breaks each person's activity change down by Microsoft 365 app, so you can see which app moved, for example Teams up while SharePoint is down. It compares the selected period against the same person's previous period of the same length.

This is a review indicator only. It is not a productivity score and not a confirmed finding. A change is often explained by normal business activity.

#### Column guide

`B_` means **Baseline** (the previous period). `C_` means **Current** (the selected period).

| Column | What it means |
|---|---|
| `UserDisplayName` | The person's display name. |
| `UPN` | The user principal name, usually their sign-in address. |
| `Workload` | The Microsoft 365 app, for example Exchange (email), SharePoint, OneDrive or Microsoft Teams. |
| `B_EventsPerDay` | Average events per day in this app during the baseline period. |
| `C_EventsPerDay` | Average events per day in this app during the selected period. |
| `ChangePct` | The percentage change in events per day for this app. Negative means less than usual; positive means more. |
| `B_Events` | Total events in this app during the baseline period. |
| `C_Events` | Total events in this app during the selected period. |
| `Status` | The review status, explained below. |

#### Status guide

| `Status` | Meaning | Action |
|---|---|---|
| `Within normal range` | This app's activity is in line with their past pattern. | No action needed. |
| `Increase - review only` | More activity in this app than usual. | Review in business context. |
| `Decrease - review only` | Less activity in this app than usual. | Review in business context. |
| `Insufficient baseline (low confidence)` | Not enough past activity in this app to compare reliably. | Treat the result carefully. |
| `No baseline activity` | No past activity in this app to compare. | Review only if current activity looks unusual. |

#### What is included

The same everyday staff activity as the headline view, split by app. Every person who is active anywhere in the period now has a row for **every** Microsoft 365 app, so an app they did not use shows as `0` instead of being missing. An app with no activity in either the baseline or the selected period reads `Status` = `Insufficient baseline (low confidence)` (there is nothing to compare). Guests, system accounts and listed service accounts are excluded.

#### How rows are sorted

Rows are grouped by person, then by app name, so you can see each person's full app picture in one block.
