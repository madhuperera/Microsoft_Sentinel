### 🧭 01. Each person compared with their own usual activity

| Workbook context | Value |
|---|---|
| Selected period | `{W_TimeRange:start}` to `{W_TimeRange:end}` |
| Baseline used for comparison | The immediately previous period of the same length |
| Normal change band | `+/- {W_NormalBandPct}%` |
| Target user filter | `{W_TargetUser}` *(blank means all users)* |

This view compares each person's Microsoft 365 activity during the selected period against their own previous period of the same length. It highlights people whose activity has risen or fallen outside the normal change band.

This is a review indicator only. It is not a productivity score and it is not a confirmed security finding. A change is often explained by leave, a role change, a project, onboarding or offboarding.

#### Column guide

`B_` means **Baseline** (the previous period). `C_` means **Current** (the selected period).

| Column | What it means |
|---|---|
| `UserDisplayName` | The person's display name. |
| `UPN` | The user principal name, usually their sign-in address. |
| `B_ActiveDayRatio` | How often they were active in the baseline period. `0.80` means active on about 80% of days. |
| `C_ActiveDayRatio` | How often they were active in the selected period. |
| `ActiveDayChangePct` | The percentage change in active days between baseline and current. Negative means active on fewer days than usual; positive means more. |
| `B_EventsPerDay` | Average activity events per day in the baseline period. |
| `C_EventsPerDay` | Average activity events per day in the selected period. |
| `EventRateChangePct` | The percentage change in events per day between baseline and current. |
| `B_Workloads` | How many Microsoft 365 apps they used in the baseline period. |
| `C_Workloads` | How many Microsoft 365 apps they used in the selected period. |
| `B_Events` | Total activity events in the baseline period. |
| `C_Events` | Total activity events in the selected period. |
| `Status` | The review status, explained below. |

#### Status guide

| `Status` | Meaning | Action |
|---|---|---|
| `Within normal range` | Activity is broadly in line with their own past pattern. | No action needed. |
| `Increase - review only` | More active than usual versus their baseline. | Review in business context. |
| `Decrease - review only` | Less active than usual versus their baseline. | Review in business context. |
| `Insufficient baseline (low confidence)` | Not enough past activity to compare reliably. | Treat the result carefully. |
| `No baseline activity` | No past activity was available to compare. | Review only if current activity looks unusual. |

#### What is included

Everyday staff activity is counted: email sent, file work, SharePoint and OneDrive use, Teams messages and changes, and meeting participation. Guests, system accounts, SharePoint system accounts and listed service accounts are excluded, so the view focuses on normal staff activity. The headline measure is active days, which is harder to inflate than a raw event count.

#### How rows are sorted

Rows show the largest fall in active days first, so the biggest drops are at the top.
