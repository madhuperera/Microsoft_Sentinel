### 🔐 04. Days signed in, compared with usual

| Workbook context | Value |
|---|---|
| Selected period | `{W_TimeRange:start}` to `{W_TimeRange:end}` |
| Baseline used for comparison | The immediately previous period of the same length |
| Normal change band | `+/- {W_NormalBandPct}%` |
| Target user filter | `{W_TargetUser}` *(blank means all users)* |

This view counts how many days each person actually signed in, and compares it with their own previous period. It is an independent cross-check on the activity views, using sign-in records rather than app activity.

This is a review indicator only. Fewer sign-in days can simply mean leave or part-time work.

#### Column guide

`B_` means **Baseline** (the previous period). `C_` means **Current** (the selected period).

| Column | What it means |
|---|---|
| `UserDisplayName` | The person's display name. |
| `UPN` | The user principal name, usually their sign-in address. |
| `B_SignInDays` | Number of days with a successful sign-in in the baseline period. |
| `C_SignInDays` | Number of days with a successful sign-in in the selected period. |
| `B_Ratio` | How often they signed in across baseline days. `0.80` means about 80% of days. |
| `C_Ratio` | How often they signed in across the selected period. |
| `ChangePct` | The percentage change in sign-in days between baseline and current. Negative means fewer than usual. |
| `B_Apps` | Number of different apps signed into during the baseline period. |
| `C_Apps` | Number of different apps signed into during the selected period. |
| `Status` | The review status, explained below. |

#### Status guide

| `Status` | Meaning | Action |
|---|---|---|
| `Within normal range` | Sign-in pattern is in line with their past. | No action needed. |
| `Increase - review only` | Signed in on more days than usual. | Review in business context. |
| `Decrease - review only` | Signed in on fewer days than usual. | Review in business context. |
| `Insufficient baseline (low confidence)` | Not enough sign-in history to compare reliably. | Treat the result carefully. |
| `No baseline activity` | No sign-in history was available to compare. | Review only if current activity looks unusual. |

#### What is included

Successful, interactive staff sign-ins from Entra ID. Listed service accounts are excluded. Background and automated sign-ins are not counted, so this reflects a person actually signing in.

#### How rows are sorted

Rows show the largest fall in sign-in activity first.
